# POE2 LootLens

**POE2 LootLens** — локальное Windows-приложение для Path of Exile 2 с двумя независимыми OCR-модулями:

1. **Оценщик комбинаций** распознаёт награды в выбранной области и показывает их рыночную стоимость поверх игры.
2. **Сканер слухов Атласа** распознаёт слухи возле корабликов, подтверждает нестабильные результаты по нескольким кадрам и хранит историю по островам.

## Основные возможности

### Оценщик комбинаций
- распознавание наград в области, выбранной пользователем;
- независимая построчная обработка OCR;
- поддержка русского и английского клиента Path of Exile 2;
- распознавание количества предметов в наборах;
- быстрый контроль количества по уже полученным gray/binary OCR-вариантам без дополнительных тяжёлых проходов;
- устойчивость к золотой подсветке строки при наведении: рамка учитывается как граница строки, а сомнительный крупный множитель временно показывается как `× ?`, не искажая итоговую цену;
- отображение стоимости в сферах возвышения или божественных сферах;
- настраиваемый порог переключения между валютами;
- отдельные состояния:
    - цена найдена;
    - известная награда без рыночной цены;
    - награда не распознана;
- несколько OCR-попыток перед отображением неизвестной награды;
- резервный блочный OCR-проход только для нераспознанных строк и точечные безопасные исправления известных стабильных ошибок OCR;
- защита от случайного открытия оверлея на обычном игровом тексте;
- автоматическое скрытие результатов после закрытия панели наград;
- кнопки «Повторить» и «Закрыть» непосредственно в оверлее;
- диагностический режим для проверки геометрии строк;
- пример правильного выбора области через кнопку i;
- поддержка пользовательских цен из custom_prices.json.

Оба модуля используют явные состояния `Stopped`, `Starting`, `Running`, `Stopping` и `Faulted`: повторные команды запуска/остановки блокируются, а ошибка запуска отображается отдельно и допускает безопасный повтор.

### Сканер слухов Атласа
- ручное сканирование по кнопке или хоткею;
- экспериментальное автоматическое сканирование при наведении;
- ручной режим используется по умолчанию;
- автоматическое определение пергаментной панели возле кораблика;
- разделение панели на три независимые строки;
- отдельная обработка каждой строки:
    - обрезка до области с глифами;
    - grayscale;
    - мягкая бинаризация;
    - усиленная бинаризация;
- резервное распознавание всей панели;
- один отдельный экземпляр Tesseract для экономии памяти;
- построчные индикаторы состояния возле игровой панели;
- отсутствие индикатора ошибки для пустых строк;
- кеширование неизменившихся успешно распознанных строк;
- повторное распознавание при изменении содержимого панели;
- подтверждение нестабильных результатов по нескольким снимкам;
- история слухов по найденным островам;
- навигация между островами;
- закрепляемый итоговый оверлей с настройкой начального закреплённого/откреплённого состояния;
- сортировка по тиру или по категории и тиру;
- режим «Сначала категория, затем тир» используется по умолчанию;
- настраиваемый порядок категорий:
    - Экспедиция;
    - Боссы;
    - Уникальные карты;
- режим геометрической диагностики с отображением найденной панели, строк, размеров и координат;
- два режима скрытия итогового оверлея:
    - вручную;
    - по таймауту.

## Хоткеи

Все глобальные хоткеи по умолчанию отключены.

Поддерживаются одиночные клавиши и сочетания с:

- Ctrl;
- Alt;
- Shift.

Для оценщика можно отдельно назначить:
- включение или выключение модуля;
- выполнить оценку сейчас;
- выбрать область;
- включить диагностический режим.

Для сканера слухов можно отдельно назначить:
- включение или выключение модуля;
- выполнить сканирование сейчас;
- включить геометрическую диагностику.

Назначенные комбинации отображаются в главном окне приложения.

## Быстрый старт
### Готовая сборка
- Скачайте архив со страницы Releases.
- Распакуйте его в отдельную папку с правом записи.
- Не запускайте приложение непосредственно из ZIP-архива.
- Запустите POE2 LootLens.exe.
- В общих настройках укажите язык клиента Path of Exile 2.
- Настройте нужный модуль.

Приложение хранит конфигурацию, пользовательские справочники и журналы рядом с EXE, поэтому не рекомендуется размещать его в защищённой от записи директории и\или удалять файлы программы.

### Запуск и трей
- при первом запуске главное окно всегда показывается;
- запуск свернутым в трей можно отдельно включить в общих настройках и он не создаёт промежуточное чёрное окно;
- кнопка свернуть отправляет приложение в трей;
- крестик закрывает приложение по умолчанию;
- при необходимости можно отдельно включить «При закрытии сворачивать приложение в трей»;
- повторный запуск ярлыка не создаёт вторую копию, а открывает уже работающее окно;
- в общих настройках можно независимо включить автозапуск оценщика и сканера слухов;
- при включении автозапуска оценщика без выбранной области приложение предупреждает, что сначала требуется калибровка.

### Настройка оценщика
1. Откройте игровое окно с наградами.
2. Нажмите «Выбрать область».
3. Выделите только список наград:
    - без внешней рамки окна;
    - без полосы прокрутки;
    - с полностью видимыми строками.
4. При необходимости откройте пример через кнопку i.
5. Запустите оценщик.
6. Откройте игровую панель наград.

### Настройка сканера слухов
1. Откройте настройки сканера слухов.
2. Выберите язык OCR, соответствующий тексту слухов.
3. Для первого знакомства оставьте ручной режим.
4. Выберите, должен ли итоговый оверлей изначально быть закреплённым; по умолчанию он откреплён.
5. Включите модуль.
6. Наведите курсор на кораблик, чтобы открыть игровую панель.
7. Нажмите «Сканировать» или назначенный хоткей.

Режим сканирования при наведении является экспериментальным.

## Рыночные данные

Приложение получает цены с внешних рыночных источников и объединяет их с локализованными названиями предметов.

Поддерживаются:

- автоматическое обновление по настраиваемому интервалу;
- ручное обновление;
- защита от параллельных запросов;
- cooldown ручной кнопки;
- сохранение предыдущего рабочего набора цен при ошибке обновления;
- пользовательские переопределения цен через `custom_prices.json`;
- цены редких Lineage Support Gems, когда категория доступна в poe.ninja;
- локальный кеш загруженных иконок.

Актуальность и полнота цен зависят от внешних источников данных.

## Приватность и сетевые запросы
OCR выполняется локально на компьютере пользователя.

### Захват экрана
Оценщик захватывает только прямоугольник, выбранный пользователем.
Сканер слухов сначала захватывает локальную область размером 960×900 вокруг курсора. Если панель не помещается в кадр, может использоваться расширенная область 1280×1200.
После обнаружения панели в OCR передаются только необходимые области изображения.

### Изображения:
- существуют только в оперативной памяти;
- не сохраняются автоматически;
- не отправляются на внешний сервер.

Оверлеи сканера слухов показываются только пока активным окном является Path of Exile 2 и скрываются при переключении на другое приложение. Они не помечаются как защищённые от экранного захвата, поэтому не должны блокировать запись NVIDIA App; при записи игры сами оверлеи могут попасть в видео.

### Сетевые запросы
Сеть используется для:

- получения рыночных цен;
- получения локализованных названий предметов;
- загрузки отсутствующих OCR-моделей;
- загрузки некоторых иконок.

Снимки экрана и распознанные изображения через сеть не передаются.

## Как сообщить об ошибке

1. При проблеме с распознаванием включите уровень логирования `Debug` и повторите ошибку.
2. Нажмите **«Диагностика ZIP»** в нижней части главного окна и выберите место сохранения.
3. Приложите созданный ZIP и скриншот игрового окна к сообщению об ошибке.
4. Для OCR-проблемы дополнительно укажите масштаб интерфейса POE2 и какая строка ожидалась.

Диагностический архив содержит версию приложения, безопасную часть настроек, состояния модулей, параметры экранов и очищенные от пользовательских путей журналы. Он не содержит снимки экрана, полный `config.json`, значения хоткеев или пользовательские файлы.

## Справочник слухов

- `rumor_catalog.default.json` — встроенные данные приложения;
- `rumor_catalog.user.json` — компактный файл только с пользовательскими дополнениями, изменёнными встроенными записями и списком отключённых ID;
- кнопка «Открыть файл правок (JSON)» нужна для опытных пользователей и открывает именно `rumor_catalog.user.json`, а не весь встроенный каталог;
- изменения применяются во время работы без перезапуска;
- редактор создаёт резервную копию и не позволяет сохранить дублирующиеся ID или OCR-фразы;
- встроенные записи не удаляются: их можно отключить, увидеть внизу списка и включить обратно;
- новые встроенные OCR-алиасы автоматически объединяются со старыми пользовательскими правками.

### Пользовательский файл содержит только:
- добавленные записи;
- изменения встроенных записей;
- список отключённых ID.

### Визуальный редактор справочника поддерживает:
- создание записей;
- автоматическую генерацию ID;
- русские и английские поля;
- OCR-алиасы;
- категории и тиры;
- теги;
- проверку обязательных полей;
- поиск дублирующихся ID;
- поиск дублирующихся OCR-фраз;
- создание резервной копии;
- отключение и повторное включение встроенных записей.

Изменения применяются без перезапуска приложения.

Новые встроенные OCR-алиасы объединяются с существующими пользовательскими правками.

Описание: [RUMOR-CATALOG.md](RUMOR-CATALOG.md).

## Журналы

Журналы находятся в каталоге `logs` рядом с EXE:

- `price-scan.log` — оценщик комбинаций;
- `rumors.log` — сканер слухов.

Размер и число файлов ограничены ротацией. Уровень логирования задаётся в общих настройках.
При уровне `Debug` файлы содержат исходный OCR и диагностику проходов. Приложение собирается как `WinExe`, поэтому при обычном запуске отдельного консольного окна нет: используйте кнопку «Открыть логи» или сформируйте готовый архив кнопкой «Диагностика ZIP». При запуске из терминала или под отладчиком Debug/Trace также зеркалируются в доступный вывод.

## Известные ограничения
### Идентификация островов

В версии 0.9.0-beta остров определяется по позиции курсора возле кораблика с небольшим радиусом.
Объединение островов только по одинаковому содержимому отключено, поскольку разные острова могут содержать одинаковые слухи.
Поиск самой иконки кораблика и отслеживание смещения карты требуют дополнительного набора эталонных кадров и пока не используются.

### Автоматический режим слухов
Режим «При наведении» является экспериментальным.
Для наиболее предсказуемой работы рекомендуется ручной режим.

## OCR

Точность OCR может зависеть от:
- разрешения экрана;
- масштаба Windows;
- масштаба интерфейса POE2;
- положения игрового окна;
- видимости всей строки;
- контрастности и фоновой текстуры;
- языка выбранной OCR-модели.

При проблемах используйте диагностический режим соответствующего модуля.

## Происхождение

Первоначальная идея происходят из форка `pedro-quiterio/PoeAncientsPriceHelper`. Текущее приложение построено на том же стеке, но совсем по другому и развивается как отдельный продукт **POE2 LootLens**.

## Технологии

.NET 8 (`net8.0-windows`), WPF, WinForms layered overlays, Tesseract OCR, MahApps.Metro, SharpHook, Newtonsoft.Json.

## Шрифты интерфейса

Интерфейс поддерживает локально встроенный Roboto без установки шрифта в Windows.
Поместите `Roboto-Regular.ttf`, `Roboto-Medium.ttf`, `Roboto-SemiBold.ttf` и
`Roboto-Bold.ttf` в `src/POE2LootLens/Assets/Fonts` перед сборкой. Проект включает их
в ресурсы приложения. Если файлов нет, автоматически используется Segoe UI.

Распределение начертаний:

- Regular — основной текст и поля ввода;
- Medium — компактные подписи;
- SemiBold — кнопки и заголовки секций;
- Bold — заголовки окон, модулей и ключевые значения.

Если файлов нет, используется системный шрифт Segoe UI.
